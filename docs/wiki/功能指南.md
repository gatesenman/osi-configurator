# 功能指南

## 字段级双向联动

聚焦左侧任意输入项，右侧 YAML/JSON 对应行即时高亮并滚动定位；点击右侧任意一行，自动切换分区、展开卡片并精确高亮到具体输入框。

![字段级双向联动](../screenshots/field-level-sync.png)

## 语义级 Lint

预览底部第二条状态栏，Schema 校验之上的业务级检查，按 **错误 / 警告 / 建议** 三级：

| 级别 | 规则示例 |
| --- | --- |
| 错误 | 实体重名；关系引用的数据集或列不存在；自引用关系 |
| 警告 | 非 snake_case 命名；指标缺 ANSI_SQL 方言；关系列数不匹配 |
| 建议 | 缺少 description；核心实体未配置 ai_context |

点击任意问题联动定位到对应配置项。

## 本体关系图

侧边栏「关系图」分区：数据集为节点（含字段数与主键摘要），关系为有向边（from 多端 → to 一端，含列映射标签）。点击节点或边联动右侧规范定位。

![本体关系图](../screenshots/graph-view.png)

## 撤销重做与自动保存

- 所有编辑（表单、AI 应用、导入、快照恢复）均可撤销，连续输入 400ms 内合并为一步，栈上限 60 步
- 自动防抖保存到本机浏览器，刷新后完整恢复

## 版本快照与对比

顶栏「版本」按钮：

- **保存当前**：命名保存（如"上线前基线"）
- **对比**：与当前版本做实体级 diff（新增 / 修改 / 删除的数据集、字段、指标、关系）
- **恢复**：一键回到该版本，恢复操作本身可撤销

![版本快照](../screenshots/snapshots.png)

## 厂商扩展（custom_extensions）

OSI 是开放标准，各厂商实现有差异——在模型 / 数据集 / 字段 / 关系 / 指标任意层级挂载厂商私有配置：

- **六大厂商预设**：COMMON / SNOWFLAKE / DBT / DATABRICKS / GOODDATA / SALESFORCE，每个键带用途说明与示例值
- **模板一键插入**：识别到已知厂商时可一键填入全部常用键；未用键以快捷芯片补加
- **双模式编辑**：扁平配置用键值对表格（值自动保留 true / 数组等 JSON 类型），复杂嵌套切 JSON 源码模式
- 数据始终按官方要求序列化为 JSON 字符串

![厂商扩展编辑](../screenshots/vendor-extensions.png)

## 导入 / 导出

- 导入 `.yaml` / `.yml` / `.json` / `.osi`，反向解析回表单；非法方言值自动归一
- 导出 YAML / JSON 双格式，带语法高亮预览与行号
- 多模型支持：官方 `semantic_model` 为数组，顶部标签页可管理多个模型
